# 프로젝트 문서 업데이트

기존의 유용한 기록을 삭제하지 않고 프로젝트 문서를 업데이트한다.

## 대상 문서

- `AGENTS.md`: Codex 및 기타 코딩 에이전트를 위한 저장소 지침.
- `docs/PROJECT.md`: 아키텍처, 범위, API, 제품 결정사항.
- `docs/PRICING.md`: 비용 관련 메모.
- `docs/WBS.csv`: 일정 또는 작업 분해.
- `docs/reports/`: 날짜별 업무보고와 분석 문서.

## 절차

1. 대상 문서와 관련 소스 파일을 먼저 읽는다.
2. 현재 작업과 관련된 섹션만 수정한다.
3. 문서를 실제 코드와 일치시킨다.
4. 문서에 적는 항목이 아직 구현 전이면 planned 상태임을 표시한다.
5. 날짜는 `YYYY-MM-DD` 형식을 사용한다.
6. 변경한 파일과 변경 이유를 요약한다.

## 스타일

- 한국어 보고서는 간결하고 결과 중심으로 작성한다.
- 기술 문서는 사실 기반으로 구체적으로 작성한다.
- 문서 전체를 갈아엎기보다 필요한 부분만 작게 수정한다.
- 저장소에서 검증하지 않은 API, 테스트, 배포 관련 주장은 추가하지 않는다.
