# 구현 계획 수립

구현 전에 짧고 실행 가능한 계획을 만든다.

## 절차

1. 관련 코드, 설정, 문서를 확인한다.
2. 현재 동작과 제약을 정리한다.
3. 작업을 독립적으로 검증 가능한 단계로 나눈다.
4. 각 단계마다 예상 변경 파일을 포함한다.
5. 리스크와 검증 명령어를 적는다.
6. 사용자가 계획 저장을 원하면 `docs/` 아래에 명확한 파일명으로 저장한다.

## 출력 형식

```markdown
## [주제] 구현 계획

### 배경
- 왜 이 작업이 필요한지

### 단계별 계획
1. **단계명** - 설명
   - 변경 파일: ...
   - 핵심 작업: ...

### 리스크/주의사항
- ...

### 완료 기준
- ...

### 검증
- ...
```

## 원칙

- 계획은 바로 실행할 수 있을 만큼 짧고 구체적으로 작성한다.
- 기존 FastAPI, Flutter, 문서 패턴을 우선 따른다.
- 실제 복잡도를 줄이는 경우가 아니면 새 모듈을 만들지 않는다.
- 사용자가 계획만 요청했다면 계획 작성 후 바로 코딩을 시작하지 않는다.
